# 🤖 AI와 함께 설치하기

Claude Code 또는 다른 AI 코딩 어시스턴트에게 이 저장소를 보여주고 설치를 요청할 수 있습니다.

---

## 🎯 AI에게 요청하는 방법

### 1단계: 저장소 공유

AI 채팅에 다음과 같이 입력하세요:

```
이 GitHub 저장소를 Google Apps Script로 배포하고 싶어요:
https://github.com/plusiam/story-worksheet-classroom

설치 과정을 도와주세요.
```

### 2단계: AI가 도와줄 수 있는 것

AI는 다음 작업을 도와줄 수 있습니다:

✅ **Google Apps Script 프로젝트 생성**
- 새 스프레드시트 생성
- Apps Script 편집기 열기
- 파일 구조 설정

✅ **코드 복사 및 설정**
- 각 `.gs` 파일과 `.html` 파일 복사
- `appsscript.json` 설정 적용
- 시트 초기화 스크립트 실행

✅ **웹앱 배포**
- 배포 설정 안내
- 권한 승인 과정 설명
- URL 확인 및 테스트

✅ **문제 해결**
- 오류 메시지 해석
- 권한 문제 해결
- 배포 이슈 디버깅

---

## 📋 AI에게 제공할 정보

설치 과정에서 AI가 다음 정보를 요청할 수 있습니다:

### 필수 정보
- Google 계정 이메일
- 학교명 (예: 서울초등학교)
- 학급명 (예: 4학년 2반)
- 선생님 성함

### 선택 정보
- 학생 명단 (CSV 또는 텍스트)
- 환영 메시지 커스터마이징
- 테마 색상 선택

## 🔧 주요 설치 단계 (AI와 함께)

### 1. 스프레드시트 준비
```
AI: "새 Google 스프레드시트를 만들어주세요"
→ drive.google.com에서 "새로 만들기 > Google 스프레드시트" 클릭
```

### 2. Apps Script 편집기 열기
```
AI: "확장 프로그램 > Apps Script를 클릭해주세요"
→ 스프레드시트에서 메뉴 클릭
```

### 3. 파일 복사
```
AI: "이제 각 파일의 코드를 복사하겠습니다..."
→ AI가 제공하는 코드를 복사/붙여넣기
```

### 4. 배포
```
AI: "배포 > 새 배포를 클릭하고..."
→ AI 안내에 따라 설정
```

## ⚠️ 자주 발생하는 이슈

### 권한 오류
```
AI에게: "Apps Script 권한 승인 화면이 나타났어요"
→ AI가 승인 과정 안내
```

### 시트 초기화 실패
```
AI에게: "STUDENTS 시트가 생성되지 않았어요"
→ AI가 수동 초기화 스크립트 제공
```

### 배포 URL 오류
```
AI에게: "웹앱 URL에 접속하면 오류가 나요"
→ AI가 디버깅 도구 활용법 안내
```

## 💡 유용한 팁

### AI 활용 극대화
- 스크린샷 공유하면 더 정확한 도움 가능
- 오류 메시지 전체를 복사해서 공유
- 단계별로 진행하고 확인받기

### 백업
```
AI에게: "설치 전에 백업 방법을 알려주세요"
→ 스프레드시트 "파일 > 사본 만들기" 안내
```

### 테스트
```
AI에게: "설치 후 테스트는 어떻게 하나요?"
→ 테스트 학생 계정 생성 및 로그인 테스트 안내
```
